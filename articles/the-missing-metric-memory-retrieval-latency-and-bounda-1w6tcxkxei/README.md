# aetna-research-lab
Benchmark and audit harness for Memory Retrieval Latency and Boundary Audit (MRLBA) of product AI agent memory.
## Article
The Missing Metric: Memory Retrieval Latency and Boundary Failures in Product AI Agents
Trustworthy AI agents are defined not by the amount they remember, but by whether every retrieval is faithful, instant, and provable across boundaries.
## Purpose
The Article proposes MRLBA not just as a concept, but as a benchmark protocol and audit harness for agent memory safety. A public code artifact makes the proposal credible, lets teams test/compare stacks, and provides minimal, inspectable benchmark scaffolding, a direct research-to-product bridge.

## Generated Notes
This repository provides reference code for the MRLBA protocol, as described in the article [The Missing Metric: Memory Retrieval Latency and Boundary Failures in Product AI Agents](https://twitter.com/aetna000).

MRLBA tests whether agent memory recall is fast, provenance-linked, and boundary-safe, not just how much memory is stored. It benchmarks recall latency, provenance, and user/session boundary errors using realistic, adversarial workflows.

## Features
- Simulate multi-user/multi-session agent workflows
- Issue retrieval probes (in-scope, out-of-scope, and distractors)
- Log what was recalled, retrieval latency (ms), provenance, and boundary violations
- Replayable, stepwise audit trace

## Usage
- Run `example_run.py` to simulate an agent retrieval step and log audit outputs
- See `mrlba_harness.py` for the protocol logic
- Benchmark outputs are JSON lines for each retrieval step: latency, provenance, and boundary tags

## Example
```bash
python example_run.py
```
See `data/audit_trace.jsonl` for a sample output.

## Files
- `README.md`: Explains the purpose/usage of the benchmark and protocol for new users and research readers.
- `mrlba_harness.py`: Implements the core MRLBA protocol logic: agents receive retrieval prompts, logging latency, provenance, and scope/boundary violations.
- `example_run.py`: Minimal runnable entry point that instantiates a harness, runs a sample adversarial session, and logs the audit output.
- `data/audit_trace.jsonl`: Example audit-trace output for review. This file is generated by running the harness with example_run.py.
## Status
Research prototype. It is public for inspection, critique, and reuse. It should not be treated as production safety proof without independent validation.
